# When to use Redis

The in-memory `RateLimiter` keeps state in a single Node process. If you run **several instances** of your app (e.g. multiple servers or containers), each has its own queue and limits. To share one limit across all of them, use **DistributedRateLimiter** with Redis.

Example: you want “at most 100 API calls per minute” across 5 servers. With Redis, all 5 share the same counter. With in-memory only, each server could do 100 per minute on its own.

**Quick comparison**

| | RateLimiter | DistributedRateLimiter |
|--|-------------|-------------------------|
| State | In one process | In Redis (shared) |
| Use when | Single server or per-process limits are fine | Multiple servers, one shared limit |
| Dependency | None | Redis (ioredis) |
